# Web Creation Skill

このスキルは、Antigravityを使用してWebサイトやLP（ランディングページ）を効率的に制作するための手順書です。

## 前提条件
- 成果物: 静的HTML/CSS (Next.js等のフレームワークは使用しない)
- 画像: ダミー画像（`picture_1`等）を使用し、後でユーザーが差し替える

## ワークフロー

### Step 1: 企画・構成の策定
以下の12のポイントを押さえて企画・構成を行う。

1. **ファーストビュー（Above the Fold）**
   - **ヘッドライン**: ベネフィット訴求、心理トリガー（緊急性・好奇心・具体性・ペインポイント）を活用
   - **サブヘッドライン**: メインの価値を補足・明確化
   - **ビジュアル**: 商品画像・動画・結果を示すイラストなど
   - **CTA（行動喚起）ボタン**: アクション志向のコピーで目立つ位置に配置
   - **ナビゲーション**: 最小限に（フルメニューは排除）

2. **ページ全体の構成**
   - **ヒーローセクション**: ヘッドライン + サブヘッド + ビジュアル + CTA
   - **社会的証明**: ロゴ、推薦の声、実績数値、受賞歴
   - **ソリューション説明**: スキャンしやすいコピー + サポートビジュアル
   - **追加CTA**: スクロールの自然な区切りに配置
   - **信頼シグナル**: セキュリティ、保証などの安心材料

3. **コピーライティング**
   - **ヘッドライン**: 広告/流入元のコピーと一致させる（メッセージマッチ）
   - **緊急性**: 「4/1までなら50%OFF」
   - **好奇心**: 「あなたのサイトを遅くしている原因とは？」
   - **具体的ベネフィット**: 「技術的SEO問題を数分で発見・修正」
   - **ペインポイント**: 「遅いサイトで顧客を失っていませんか？」

4. **CTA最適化**
   - 汎用的な「送信」ではなくベネフィット訴求 + 一人称のラベル（例：「無料で始める」）
   - コントラストの高いボタン色
   - ファーストビューに配置 + スクロール途中にも繰り返し配置
   - 視線を誘導する視覚的な手がかりを活用

5. **フォーム設計**
   - 入力項目は最小限に（初回は必要最低限のみ）
   - プログレッシブ・プロファイリング（段階的に情報取得）
   - マルチステップフォームで体感負担を軽減
   - 明確なエラーメッセージ
   - プライバシー保証の表示（個人情報入力欄の近く）
   - モバイル向け最適化（メールキーボード、数字パッドなど）

6. **信頼性・社会的証明**
   - 顧客ロゴ（有名企業）
   - ターゲットに合わせたテスティモニアル（お客様の声）
   - 導入実績の数値（利用者数・満足度など）
   - 受賞歴・認定バッジ
   - 外部レビュープラットフォームの評価

7. **デザイン原則**
   - **ビジュアルヒエラルキー**: 機能よりも「成果」を見せる
   - 十分な余白（特にCTA周り）
   - **ナビゲーション排除**: 離脱を防ぐためフッターリンクも最小限
   - ターゲット層に合ったビジュアルスタイル

8. **モバイル最適化**
   - レスポンシブデザイン
   - タッチフレンドリーなボタン（十分な間隔）
   - ピンチズーム不要のフォントサイズ
   - 縦積みレイアウト
   - ページ速度の最適化（画像・アセット軽量化）

9. **LPの種類（目的別）**
   - **リード獲得型**: フォームで連絡先情報を収集（B2B/SaaS向け）
   - **クリックスルー型**: 情報を伝えてからコンバージョンページへ誘導
   - **スクイーズページ**: ミニマルデザインでメールアドレスのみ取得
   - **セールスページ**: 長尺で詳細説明、高単価商品向け

10. **テスト＆改善**
    - **A/Bテスト**: 1要素ずつ変更、トラフィックを均等分割
    - **テスト対象**: ヘッドライン、コピー、ビジュアル、レイアウト、CTA、フォーム
    - **KPI管理**: CVR、直帰率、フォーム完了率

11. **SEO対策**
    - インデックスブロックされていないか確認
    - canonicalタグで重複コンテンツ管理
    - ページ速度・モバイル対応
    - HTTPS対応
    - 適切な内部リンク（ただし離脱させない）
    - 見出し構造の最適化

12. **LP作成すべきタイミング**
    - 新商品・新機能のローンチ（予約・ウェイトリスト）
    - 期間限定プロモーション
    - リードマグネット配布時
    - 異なるターゲットセグメント別にページを分ける時

### Step 2: コンテンツ作成
1. **テキストライティング**
   - 確定した構成案に基づき、各セクションの文章（ヘッドコピー、ボディコピー、オファー文など）を作成する。

### Step 3: デザイン設計
1. **ダミー画像の定義**
   - デザインに必要な画像の箇所にプレースホルダー（`picture_1.png`など）を配置する計画を立てる。
2. **フォント設計**
   - フォントの組み合わせは、「[Google Fonts フォント組み合わせ](file:///Users/bandoutakanori/Antigravity/HP/rule/font/google_fonts_combinations.md)」を参照して、商材のコンセプトに最適なものを選択する。
3. **CSS設計**
   - ユーザーの要望や商材のイメージに合わせたカラーパレット、レイアウト方針を策定する。モダンで反応率の高いデザインを心がける。
   - 配色の決定方法は「[Webサイト配色基本ルール](file:///Users/bandoutakanori/Antigravity/HP/rule/color/web_site_basic_color_rules.md)」を参照する。
4. **ファビコン・OGP画像の準備**
   - サイトのブランドロゴに基づいたファビコン（`favicon.ico`等）と、SNS共有用のOGP画像（`og-image.png`等）を準備する計画を立てる。
5. **ヒーローセクションの設計**
   - ヒーローセクション（ファーストビュー）のデザイン方針は、「[ファーストビューの種類分け](file:///Users/bandoutakanori/Antigravity/HP/rule/hero%20section/fast%20view/classification.md)」を参照して決定する。

### Step 4: コーディング (実装)
1. **HTML/CSS実装**
   - `index.html`, `style.css` を作成する。
   - **静的サイト**として構築する。
   - モバイルファーストでレスポンシブ対応を行う。
   - SEOタグ（title, meta description等）を適切に設定する。
   - **ファビコンとOGPの設定**を `head` 内で行う（`link rel="icon"`, `meta property="og:*"` 等）。

### Step 5: 検証と修正
1. **ブラウザ確認**
   - 実装したページをブラウザで開き、崩れや誤字脱字がないか確認する。
   - ヒーローセクションの視認性が悪い場合は、「[FVデザインパターン](file:///Users/bandoutakanori/Antigravity/HP/rule/hero%20section/fv_design_patterns.md)」を参照して修正する。
2. **ユーザーフィードバック対応**
   - ユーザーの指摘に基づき、デザインや文言を微修正する。
3. **LP納品前のチェックリスト**
   - **1. デザイン**
     - **文字サイズ・行間**: 文字サイズや行間が適切で、ストレスなく読めるか？
     - **余白**: 要素同士が詰まりすぎて見づらくなっていないか？
     - **誤字脱字**: 誤字脱字がないか？
     - **画像**: 透かし入り画像を使っていないか、容量は適切に圧縮されているか？（ページ全体…1.6MB以内、画像1枚…200〜300KB が目安）
   - **2. フォーム周り**
     - **入力エラー**: 未入力時やエラー時に、正しくエラーメッセージが表示されるか？
     - **送信〜完了画面**: 送信後、完了画面まで問題なく遷移するか？
     - **自動返信メール**: 自動返信メールが正しい内容で届くか？
   - **3. コード・動作**
     - **レイアウト**: HTML/CSSのミスによる表示崩れが起きていないか？
     - **ブラウザ表示**: Chrome・Safari等主要ブラウザで問題なく表示されるか？
     - **リンク**: リンク切れや、意図しないページに遷移していないか？
     - **解析タグ**: Google Analyticsや広告タグが正しく設置されているか？
   - **4. headタグ・設定**
     - **title / description**: ページタイトル・ディスクリプションが正しく設定されているか？
     - **canonical**: URLの表記揺れで重複コンテンツにならない設定になっているか？
     - **favicon**: ブラウザタブにファビコンが正しく表示されているか？
     - **OGP**: SNSでシェアしたとき、画像やタイトルが正しく表示されるか？

## 実行時の注意点
- 常にユーザーと対話しながら進めること。特に構成案の段階での合意形成を重視する。
- 複雑なJavaScriptや動的フレームワークの使用は避ける（ユーザー要件による）。
