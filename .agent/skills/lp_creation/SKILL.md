# Web Creation Skill

このスキルは、Antigravityを使用してWebサイトやLP（ランディングページ）を効率的に制作するための手順書です。

## 前提条件
- 成果物: 静的HTML/CSS (Next.js等のフレームワークは使用しない)
- 画像: ダミー画像（`picture_1`等）を使用し、後でユーザーが差し替える

## AIを活用した制作の7つのベストプラクティス（原則）

AIがLP作成において高い精度の出力を出せるよう、本スキル実行時は**必ず以下の1〜7の順で、1つずつステップバイステップでユーザーに必要な資料や指定を要求**すること。
一度に複数の要求を行わず、ユーザーからの回答や資料提供を待ってから次の要求に進むこと。全ての情報の提供を受けてから下部のワークフローに進むこと。

1. **ワイヤーフレームの要求**
   - 「テキスト指示の前に、Excalidraw等で作成したざっくりとした配置（ワイヤーフレーム）のスクリーンショットを提供してください」とユーザーに依頼する。
   - ※AIは想像より模倣が得意なため、画像を使うことで意図通りのレイアウトになりやすい（手描きレベルで十分）。
   - **【最重要】提供されたワイヤーフレームのセクション構成や要素の配置順は「絶対」に遵守すること。AI側の勝手な解釈でセクションを削ったり、別の構成（例：別の参考画像の構成）で上書きしてはならない。**

2. **参考デザイン（スクリーンショット）の要求**
   - 「ナビバー、ヒーローセクション、料金カードなど、パーツ単位で参考になるUIのスクリーンショット（DribbbleやMobbin等）を提供してください」と依頼し、「このスタイルをコピーして」という指示を受け取る。
   - ※ページ全体ではなく、**パーツごと個別**に画像を受け取ること。

3. **ムードボード（配色指定）の要求**
   - 「『温かみのある感じで』といった言葉ではなく、好きな配色のWebサイトやデザインを集めたカラーパレット画像（HEXコード付き）を提供してください」と依頼する。

4. **デザインシステム（デザイントークン）の定義**
   - コードを書き始める前に、「ブランドカラー、フォント、余白（Spacing）、角丸（Border-radius）などのデザイントークンを指定してください」と依頼し、スタイルを明文化する。
   - ※これらをページ間で統一することでデザインのブレを防ぐ。

5. **アンチパターン（やるなリスト）の確認**
   - 「AI特有の『よくあるデザイン』を排除するため、禁止事項（例：AIグラデーション禁止、Interフォント禁止、Lucideアイコンのみの使用禁止など）があれば指定してください」とユーザーに確認する。

6. **修正指示におけるスクリーンショットの使用推奨**
   - 作業後、ユーザーに修正を依頼してもらう際は、「修正箇所は言葉だけでなく、ブラウザのスクショに赤丸をつけ『ここを24pxに』『border-radiusを4pxに』といった具体的な数値とともに指示してください」と案内する。

7. **デフォルト以外のフォントとアイコンの選定**
   - 「『AIが作った感』を消すため、使用するGoogle Fonts（M PLUS、BIZ UDゴシック、Poppinsなど）と、Lucide以外のアイコンライブラリ（Phosphor、Tabler、Heroiconsなど）を指定してください」と依頼する。

### ワークフロー

上記のベストプラクティスを通じて十分なヒアリングが行われた後、事前の要件に従い以下のStep 1〜5の順に従って制作を進める。

#### Step 1: 企画・構成の策定
ヒアリング内容に基づき、冗長な作業を排して以下の要素を満たす構成案を策定する。

1. **LPの目的と種類の確認**
   - リード獲得型（B2B/SaaS）、クリックスルー型、スクイーズページなど、目的に応じて最適な構成を選択する。
   - **※ユーザーから提供されたワイヤーフレームがある場合は、無条件でそのワイヤーフレームの構成要素とブロック配置を正として扱うこと。**
2. **ページ全体の構成とファーストビュー（Above the Fold）**
   - **ヒーローセクション**: ベネフィット訴求と心理トリガーを含むコピー、ビジュアル、CTAを配置する。
   - **信頼性の提示**: お客様の声や実績数値などの社会的証明をファーストビュー直下に自然に配置する。
   - **ソリューション説明**: スキャンしやすいコピーで情報を提示する。
3. **コピーライティングとCTA最適化**
   - 単なる「送信」ではなく、ベネフィット訴求力のあるアクション志向のCTA（例：「無料で始める」）にする。
   - ユーザビリティを考慮し、スクロールの区切りごとにCTAを繰り返し配置する。
4. **フォーム設計・モバイル最適化**
   - フォームの入力項目は必要最小限に抑え、多すぎる場合はプログレッシブ設計（マルチステップ）にする。
   - スマホ閲覧を前提とした縦積みレイアウト、タッチしやすいボタン間隔を確保する。

#### Step 2: コンテンツ作成
1. **テキストライティング**
   - 確定した構成案に基づき、各セクションの文章（見出し、本文など）を作成し、ページ全体のメッセージを揃える。

#### Step 3: デザイン設計
1. **ダミー画像の定義**
   - デザインに必要な箇所にプレースホルダー（`picture_1.png`など）を配置する。
2. **フォント・CSSカラー設計**
   - 事前に定義したデザイントークンに従う。指定がない場合は「[Google Fonts フォント組み合わせ](file:///Users/bandoutakanori/Antigravity/HP/rule/font/google_fonts_combinations.md)」や「[Webサイト配色基本ルール](file:///Users/bandoutakanori/Antigravity/HP/rule/color/web_site_basic_color_rules.md)」を参照する。
3. **ヒーローセクションの設計**
   - 「[ファーストビューの種類分け](file:///Users/bandoutakanori/Antigravity/HP/rule/hero%20section/fast%20view/classification.md)」を参照し、要件に合わせたFVデザインを決定する。
4. **ファビコン・OGPの準備**
   - `favicon.ico`やSNS共有用の`og-image.png`を組み込む計画を立てる。

#### Step 4: コーディング (実装)
1. **HTML/CSS実装**
   - 策定した設計をもとに静的サイト（`index.html`, `style.css`）として実装する。
   - レスポンシブ対応およびSEO対策（title, meta description, canonical, favicon, OGPなど）を盛り込む。

#### Step 5: 検証と修正
1. **ブラウザ確認とクオリティチェック**
   - ページ内の表示崩れ、誤字脱字、画像容量（全体1.6MB以内目安）を確認する。
   - ヒーローセクションなど重要箇所の視認性が悪い場合は「[FVデザインパターン](file:///Users/bandoutakanori/Antigravity/HP/rule/hero%20section/fv_design_patterns.md)」を参考に修正する。
2. **納品前チェックリストの完遂**
   - 【デザイン】文字サイズや行間、余白が適切か？
   - 【フォーム・リンク】エラー表示や正常な遷移が行われるか？
   - 【SEO周り】title, description, OGP, 解析・広告タグが正しく設置されているか？
3. **ユーザーフィードバックへの対応**
   - 定義された修正指示手順（スクショを用いた具体的な指示）により、的確に修正を反映する。

## 実行時の注意点
- 常にユーザーと対話しながら進めること。特に構成案の段階での合意形成を重視する。
- 複雑なJavaScriptや動的フレームワークの使用は避ける（ユーザー要件による）。
